# AI (KIRO) BEHAVIOR RULES

## Default Mode

- Deterministic / spec-driven
- No exploratory changes unless requested

---

## What AI MUST NOT Do

- Introduce new frameworks
- Change architecture
- Modify unrelated files
- Optimize beyond requirements

---

## Explain Before Acting

For any change affecting:
- Core logic
- Shared modules
- Interfaces

AI MUST explain impact before generating code.
